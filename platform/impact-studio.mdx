Impact Studio is the module authoring surface used by the frontend and backend tool APIs.

## What Studio Handles

- Create/edit module definitions and components
- Run tool-assisted drafting workflows
- Persist module changes to the data layer

## Current Tool Endpoints

Studio exposes the following backend routes:

- `POST /api/v1/tools/deep_research`
- `POST /api/v1/tools/scoring_preview`
- `POST /api/v1/module_management/store_module`

## Tool Capabilities

### Deep Research

- Input: current module UI content
- Behavior: generates a research report with web-search assistance
- Output: structured report payload

### Scoring Preview

- Input: module UI content + sample conversation prompt
- Behavior: simulates scoring-level outputs
- Output: `high`, `baseline`, `medium`, `low` response variants

### Store Module

- Input: module UI content
- Behavior: create/update module and synchronize components
- Output: stored module entity

## How It Fits in the Workflow

1. Draft module in Studio
2. Use deep research and scoring preview to refine rubric/design
3. Save module
4. Activate module for eval/lens processing
5. Observe results in Analyse/Verify

## Service Dependencies

Studio tools currently rely on:

- semantic-kernel-based orchestration in `impact-studio`
- configured model services (regular/reasoning)
- optional Google web-search plugin credentials
