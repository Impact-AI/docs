Intelligence is Impact AI's analytics engine. It processes evaluation results to surface metrics, trends, insights, and alerts about your AI product's performance.

## Overview

The Intelligence system:

- **Aggregates** evaluation results into metrics
- **Analyzes** trends and patterns over time
- **Generates** AI-powered insights
- **Triggers** alerts when metrics cross thresholds

## Metrics

Metrics are quantitative measurements derived from module evaluations.

### Metric Types

| Metric | Description |
|--------|-------------|
| **Result** | Primary aggregated score (avg, count, distribution) |
| **Flag** | Boolean indicator for pass/fail criteria |
| **Alert** | Triggered when thresholds are crossed |
| **Confidence** | Aggregated evaluation confidence |
| **Interactions** | Count of evaluated interactions |

### Result Metric

The Result metric varies by module type:

**Impact Eval:**
```json
{
  "value": 4.2,
  "change": 0.3,
  "aggregates": {
    "avg": 4.2,
    "min": 2,
    "max": 5,
    "count": 1250
  },
  "histogram": {
    "1": 12,
    "2": 45,
    "3": 180,
    "4": 520,
    "5": 493
  },
  "timeseries": {
    "3600": [...],
    "86400": [...]
  }
}
```

**Impact Lens:**
```json
{
  "tags": [
    {"name": "product_inquiry", "value": 450, "change": 12.5},
    {"name": "support_request", "value": 380, "change": -5.2},
    {"name": "feedback", "value": 120, "change": 8.1}
  ]
}
```

### Aggregations

Metrics support multiple aggregation methods:

| Aggregation | Description |
|-------------|-------------|
| `avg` | Average value |
| `sum` | Total sum |
| `count` | Number of items |
| `min` / `max` | Minimum/maximum values |
| `p50` / `p95` / `p99` | Percentiles |

### Time Windows

View metrics across different time windows:

- **1 hour** — Real-time monitoring
- **24 hours** — Daily patterns
- **7 days** — Weekly trends
- **30 days** — Monthly overview
- **All time** — Complete history

## Insights

Insights are AI-generated observations about your product's performance.

### Insight Types

| Type | Description |
|------|-------------|
| **Trend** | Improving, degrading, or stable patterns |
| **Anomaly** | Unusual patterns or outliers |
| **Opportunity** | Areas for improvement |
| **Correlation** | Related metrics or behaviors |

### Example Insights

```json
{
  "insights": [
    {
      "type": "trend",
      "value": "Response helpfulness improved 15% this week",
      "rationale": "Score increased from 3.8 to 4.4 average"
    },
    {
      "type": "anomaly",
      "value": "Unusual spike in hallucination flags",
      "rationale": "50% increase on Tuesday correlates with model update"
    },
    {
      "type": "opportunity",
      "value": "Technical queries underperforming",
      "rationale": "Technical topic scores 25% lower than general queries"
    }
  ]
}
```

### Summaries

Each module can have an AI-generated summary:

> "Response helpfulness has been consistently high this week, averaging 4.2/5. The main areas for improvement are technical queries, which score lower due to incomplete information. Consider enhancing the knowledge base for technical topics."

## Alerts

Alerts notify you when metrics cross defined thresholds.

### Configuring Alerts

Set alerts on any module:

```json
{
  "alertable": true,
  "alert_config": {
    "threshold": 3.5,
    "direction": "below",
    "window": "24h",
    "sensitivity": "medium"
  }
}
```

### Alert States

| State | Description |
|-------|-------------|
| **Active** | Currently triggered |
| **Resolved** | Previously triggered, now normal |
| **Muted** | Temporarily silenced |

### Viewing Alerts

Access alerts from:
- **Dashboard** — Alert indicators on metrics
- **Alerts Panel** — Dedicated alert management
- **Notifications** — Email/webhook notifications

## Scopes

Intelligence operates at different scopes:

### Module Scope

Detailed metrics for a single evaluation module:
- Full metric breakdown
- Summary and insights
- Historical trends

### Widget Scope

Aggregated metrics across modules in a widget:
- Combined scores
- Cross-module patterns

### Page Scope

High-level metrics for a page category:
- Summary metrics
- Page-level trends

### Product Scope

Overall product health:
- Interaction counts
- Global trends
- Key performance indicators

## Filters

Filter intelligence data by:

| Filter | Description |
|--------|-------------|
| **Timeframe** | Time window for aggregation |
| **Modules** | Specific modules to include |
| **Result Predicates** | Filter by result values |

### Example Filter

```json
{
  "timeframe": 2592000,
  "modules": [
    {
      "module_id": "helpfulness",
      "result_predicates": [{"op": "gte", "value": 3}]
    }
  ]
}
```

## Real-Time Updates

Intelligence updates in real-time as new traces flow in:

1. Traces arrive via SDK
2. Modules evaluate the interaction
3. Results update cached metrics
4. Dashboard reflects changes
5. Alerts trigger if thresholds crossed

Updates propagate within seconds of receiving traces.
