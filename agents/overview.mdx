## How Agents Work

Agents are AI assistants powered by large language models with access to Impact AI's platform capabilities. They:

1. **Understand context** — Access your product configuration, data, and history
2. **Take actions** — Create, configure, and manage resources
3. **Provide guidance** — Offer recommendations based on best practices
4. **Automate workflows** — Handle multi-step processes end-to-end

## Interacting with Agents

### Chat Interface

Each agent is accessible via chat. Navigate to the relevant section and start a conversation:

```
You: I want to set up evaluation for my customer support bot

Max: I'll help you configure evaluation modules. Based on your 
product, I recommend starting with:

1. Response Helpfulness - Does the response address the query?
2. Tone Appropriateness - Is the tone professional?
3. Resolution Detection - Was the issue resolved?

Should I create these modules for you?
```

## Agent Specializations

### Laila — The Controller

<img src="/images/agent_0_laila.png" width="48" />

Laila handles onboarding and governance:

- **Product Setup** — Guides you through adding new products
- **Configuration** — Helps configure connections and settings
- **Compliance** — Ensures adherence to policies
- **Documentation** — Manages product documentation

Best for: Getting started, compliance questions, product setup

### Max — The Manager

<img src="/images/agent_1_max.png" width="48" />

Max handles product management:

- **Module Configuration** — Creates and configures evaluation modules
- **Auto-Generation** — Generates modules from product context
- **Lifecycle Management** — Manages product and version lifecycle
- **Prioritization** — Helps prioritize features and improvements

Best for: Setting up evaluation, managing modules, product strategy

### Iris — The Evaluator

<img src="/images/agent_5_iris.png" width="48" />

Iris handles testing and validation:

- **Simulations** — Runs automated test scenarios
- **Synthetic Users** — Generates test users and conversations
- **User Personas** — Creates representative user profiles
- **Quality Validation** — Tests product quality systematically

Best for: Testing, validation, quality assurance

### Orion — The Analyst

<img src="/images/agent_7_orion.png" width="48" />

Orion handles analytics and insights:

- **Data Analysis** — Analyzes trace and evaluation data
- **Trend Detection** — Identifies patterns and trends
- **Insight Generation** — Creates actionable insights
- **Impact Assessment** — Measures impact of changes

Best for: Understanding performance, finding insights, data exploration

### Sage — The Aligner

<img src="/images/agent_6_sage.png" width="48" />

Sage handles module building:

- **Module Design** — Helps design evaluation criteria
- **Rubric Creation** — Creates scoring rubrics
- **Configuration** — Configures module settings
- **Alignment** — Ensures modules match business goals

Best for: Building modules in Impact Studio, refining evaluations

## Best Practices

<AccordionGroup>
  <Accordion title="Be specific in requests">
    Clear requests get better results:
    
    ```
    ✓ "Create a module to evaluate if responses include 
       source citations when answering factual questions"
    
    ✗ "Make an evaluation module"
    ```
  </Accordion>
  
  <Accordion title="Use the right agent">
    Match your task to the agent's specialty:
    
    - Setup questions → Laila
    - Module configuration → Max or Sage
    - Testing → Iris
    - Analytics → Orion
  </Accordion>
  
  <Accordion title="Iterate and refine">
    Agents work well with iterative refinement:
    
    1. Start with initial request
    2. Review agent's output
    3. Provide feedback
    4. Refine together
  </Accordion>
</AccordionGroup>
