## Simulations

Iris runs automated simulations to test your AI product:

### Creating a Simulation

```
You: Create a simulation to test edge cases

Iris: I'll help you create a simulation for edge case testing.

**Simulation Setup:**

What types of edge cases do you want to test?

1. Adversarial inputs (prompt injection, jailbreaks)
2. Out-of-scope questions
3. Ambiguous queries
4. Multi-turn complexity
5. Language/format variations

You: Focus on out-of-scope questions and ambiguous queries

Iris: Got it. I'll create a simulation with:

- 50 out-of-scope questions (topics outside your product's domain)
- 50 ambiguous queries (unclear intent, multiple interpretations)

**Configuration:**
- Total interactions: 100
- Persona: Mixed (new and experienced users)
- Expected behavior: Graceful handling, clarification requests

Ready to run?
```

### Running Simulations

```
You: Run the simulation

Iris: Starting simulation...

Progress: ████████████░░░░░░░░ 60%

**Live Results:**
- Completed: 60/100 interactions
- Avg response time: 1.2s
- Preliminary quality score: 4.1/5

Early observations:
- Out-of-scope handling: Good (redirects appropriately)
- Ambiguous queries: Mixed (asks for clarification 70% of time)

Continuing...
```

### Simulation Results

```
Iris: Simulation complete!

**Summary:**
- Total interactions: 100
- Success rate: 87%
- Avg quality score: 4.0/5

**Findings:**

✓ **Out-of-scope handling** (Score: 4.3/5)
  - Correctly identifies off-topic questions
  - Provides helpful redirects

⚠ **Ambiguous query handling** (Score: 3.7/5)
  - Sometimes guesses instead of clarifying
  - 30% of ambiguous queries got wrong interpretation

**Recommendations:**
1. Add clarification prompt for low-confidence queries
2. Expand out-of-scope response templates

View detailed results?
```

## User Personas

Iris creates detailed personas for targeted testing:

### Creating Personas

```
You: Create a persona for frustrated customers

Iris: Creating "Frustrated Customer" persona:

**Profile:**
- Emotional state: Frustrated, impatient
- Communication: Short messages, may include caps
- Expectations: Quick resolution, empathy
- Trigger topics: Billing issues, long wait times

**Example behaviors:**
- "This is the THIRD time I'm asking about this"
- "Why is this so complicated?"
- "I just want my money back"

**Test scenarios:**
1. Repeated issue after previous contact
2. Long resolution time complaint
3. Threatening to switch competitors

Create interactions with this persona?
```

### Persona Library

Build a library of personas:

| Persona | Description | Test Focus |
|---------|-------------|------------|
| Happy Customer | Satisfied, easy-going | Baseline quality |
| Frustrated Customer | Impatient, upset | Empathy, de-escalation |
| Confused Customer | Unclear needs | Clarification, patience |
| Technical Expert | Detailed questions | Accuracy, depth |
| New User | First-time user | Onboarding, clarity |

## Best Practices

<AccordionGroup>
  <Accordion title="Test regularly">
    Run simulations:
    - Before major releases
    - After prompt changes
    - Weekly for ongoing monitoring
  </Accordion>
  
  <Accordion title="Cover diverse scenarios">
    Include tests for:
    - Happy paths (expected usage)
    - Edge cases (unusual inputs)
    - Error conditions (failures, timeouts)
    - Different user types
  </Accordion>
  
  <Accordion title="Act on findings">
    Use simulation results to:
    - Identify improvement areas
    - Prioritize fixes
    - Validate changes
  </Accordion>
</AccordionGroup>
