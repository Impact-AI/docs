This section replaces older agent-first documentation with the current service/tool architecture.

## Core Evaluation Stack

| Service | Responsibility |
|---|---|
| `impact-telemetry` | OTLP ingest (`/v1/traces`, `/v1/logs`) |
| `impact-metric` | Span normalization and trace/interaction metric processing |
| `impact-eval` | Event-driven score generation for eval modules |
| `impact-lens` | Event-driven tagging/classification for lens modules |
| `impact-issue` | Diagnosis and issue aggregation endpoints |

## Authoring and Tooling

| Service | Responsibility |
|---|---|
| `impact-studio` | Deep research, scoring preview, module storage |
| `admin` | Workspace, membership, onboarding, API keys, product/version setup |

## Synthetic Data and Testing Services

| Service | Responsibility |
|---|---|
| `impact-persona` | Persona generation |
| `impact-synt` | Artificial user generation |
| `impact-simulate` | Interaction simulation generation |

## Event-Driven Pattern

Several services subscribe to new trace events and process asynchronously. This supports scalable ingestion and avoids blocking your app path.

## Primary Service Endpoints

- `impact-telemetry`: `POST /v1/traces`, `POST /v1/logs`
- `impact-studio`: `POST /api/v1/tools/deep_research`, `POST /api/v1/tools/scoring_preview`
- `impact-eval`: `GET /health` (runtime driven by subscriptions)
- `impact-lens`: `POST /tag_interaction/`, `POST /store/tag_interaction/`
- `admin`: workspace, key, invite, setup, and document routes
