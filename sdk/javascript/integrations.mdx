## Enabling Opt-In Integrations

LangChain and Azure Foundry Agents are disabled by default. Enable them explicitly:

```typescript
impact.init({
  apiKey: process.env.IMPACT_API_KEY,
  instrumentations: {
    langchain: true,
    foundryAgents: true,
  },
});
```

LangChain also requires a separate package install:

```bash
npm i @traceloop/instrumentation-langchain
```

## Disabling Specific Integrations

```typescript
impact.init({
  apiKey: process.env.IMPACT_API_KEY,
  instrumentations: {
    anthropic: false,
    pinecone: false,
  },
});
```

<AccordionGroup>
  <Accordion title="Checking Instrumentation Status">
    After initialization, inspect which instrumentations loaded successfully:

    ```typescript
    impact.init({ apiKey: process.env.IMPACT_API_KEY });

    for (const result of impact.instrumentationResults) {
      console.log(`${result.displayName}: ${result.code} â€” ${result.message}`);
    }
    ```

    Each result includes:

    | Field | Description |
    |---|---|
    | `key` | Instrumentation identifier (e.g., `"openai"`, `"anthropic"`). |
    | `displayName` | Human-readable name. |
    | `code` | Status: `enabled`, `disabled`, `module_not_found`, `module_load_error`, etc. |
    | `message` | Descriptive status message. |
    | `layer` | `model`, `agent`, `tooling`, or `vector_db`. |
  </Accordion>

  <Accordion title="Important Caveats">
    - Coverage depends on upstream instrumentation packages.
    - Some frameworks emit richer events when OTLP logs are enabled (`logsEnabled: true`).
    - Dependency conflicts are handled as non-fatal skips.
  </Accordion>
</AccordionGroup>
