Auto-instrumentation captures provider calls. Use `@trace` for your own workflow steps.

## Decorator API

```python
from impact import trace

@trace
@trace(type="workflow")
@trace(type="tool", name="search_docs", attributes={"index": "products"})
def fn(...):
    ...
```

## Parameters

| Parameter | Description |
|---|---|
| `type` | Span classification string (default: `"span"`) |
| `name` | Optional span name override (default: function qualname) |
| `attributes` | Custom attributes stored as `impact.trace.<key>` |

Reserved suffixes are protected (`type`, `name`, `path`, `input`, `output`).

## Captured Fields

The decorator sets:

- `impact.trace.type`
- `impact.trace.name`
- `impact.trace.path`
- `impact.trace.input`
- `impact.trace.output`

Errors are recorded on the span and status is set to `ERROR`.

## Nested Example

```python
from impact import trace

@trace(type="workflow", name="chat_request")
def handle_chat(message: str) -> str:
    context = retrieve_context(message)
    return generate_answer(message, context)

@trace(type="retrieval")
def retrieve_context(query: str):
    ...

@trace(type="task")
def generate_answer(query: str, context):
    ...
```

## Async and Streaming Support

`@trace` supports:

- sync functions
- async functions
- generators
- async generators
