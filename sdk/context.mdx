Use `impact.context()` to enrich all spans in the current execution context.

## Function

```python
impact.context(
    user_id: str | None = None,
    interaction_id: str | None = None,
    version_id: str | None = None,
    **extras: Any,
)
```

## Stored Attributes

Context fields are emitted as span attributes under `impact.context.*`.

- `impact.context.user_id`
- `impact.context.interaction_id`
- `impact.context.version_id`
- `impact.context.<extra_key>`

## Basic Example

```python
import impact

impact.context(
    user_id="user_123",
    interaction_id="conv_456",
    plan="enterprise",
    region="eu",
)
```

## Request Middleware Example (FastAPI)

```python
from fastapi import FastAPI, Request
import impact

impact.init()
app = FastAPI()

@app.middleware("http")
async def add_impact_context(request: Request, call_next):
    impact.context(
        user_id=request.headers.get("X-User-Id"),
        interaction_id=request.headers.get("X-Session-Id"),
    )
    return await call_next(request)
```

## Behavior Notes

- Multiple calls merge values in current context.
- Later calls override duplicate keys.
- Context propagates across async tasks and instrumented threaded flows.

## Best Practices

- Set `interaction_id` consistently across all turns in one conversation.
- Always set `user_id` (or stable anonymous surrogate).
- Add low-cardinality business metadata where useful for filtering.
