The Python SDK is the primary integration path for Impact AI.

Package name: `impact`

## What It Does

- Sends traces and logs to Impact over OTLP HTTP
- Auto-instruments supported AI providers/frameworks
- Attaches `impact.context.*` attributes across execution context
- Provides a single manual span decorator: `@trace`
- Works in BYO OpenTelemetry setups (`auto`, `attach`, `bootstrap`)

## Minimal Integration

```python
import impact
from openai import OpenAI

impact.init()
impact.context(user_id="u_123", interaction_id="int_456")

client = OpenAI()
client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "Hello"}],
)
```

## Public API

```python
import impact

impact.init(...)
impact.context(...)
impact.trace(...)
impact.instrument_asgi_app(app)
impact.shutdown()
```

## Runtime Modes

| Mode | Behavior |
|---|---|
| `auto` (default) | Attach to existing tracer provider when available, otherwise bootstrap |
| `attach` | Require existing provider and add Impact processors/exporters |
| `bootstrap` | Create and register a new provider |

## Endpoint Resolution

By default, `impact.init()` derives endpoint routing from `IMPACT_API_KEY` (`impact_<region>_*` format).

For self-hosted/custom deployments, override endpoint with:

- `api_endpoint` in `impact.init(...)`, or
- `IMPACT_BASE_URL` environment variable.

## Key Python-Specific Points

- `IMPACT_API_KEY` is required (argument or env var)
- SDK sets GenAI semantic-convention defaults for compatible instrumentations
- Context is stored via `contextvars`, so it works with async/threaded flows
- `shutdown()` flushes and stops Impact-owned processors/exporters
