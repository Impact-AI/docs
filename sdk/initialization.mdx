The `init()` function configures the Impact AI SDK and starts telemetry collection.

## Basic Usage

```python
from impact.sdk import init

init()
```

This reads configuration from environment variables `IMPACT_API_KEY` and `IMPACT_BASE_URL`.

## Parameters

```python
init(
    api_key: str | None = None,
    api_endpoint: str | None = None,
    mode: Literal["auto", "bootstrap", "attach"] = "auto",
    disable_batch: bool = False,
    disable_logs: bool = False,
)
```

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `api_key` | `str` | `None` | API key for authentication. Falls back to `IMPACT_API_KEY` env var. |
| `api_endpoint` | `str` | `None` | Impact AI endpoint URL. Falls back to `IMPACT_BASE_URL` env var. |
| `mode` | `str` | `"auto"` | Initialization mode (see below) |
| `disable_batch` | `bool` | `False` | Disable batching for immediate export |
| `disable_logs` | `bool` | `False` | Disable log capture (GenAI message content) |

## Initialization Modes

The `mode` parameter controls how the SDK interacts with existing OpenTelemetry setups:

### Auto Mode (Default)

```python
init(mode="auto")
```

- Checks if a TracerProvider already exists
- If yes: attaches to it (your existing OTel setup continues working)
- If no: bootstraps its own TracerProvider

This is the safest option and works with existing observability setups.

### Bootstrap Mode

```python
init(mode="bootstrap")
```

- Always creates and sets a new TracerProvider
- Replaces any existing TracerProvider
- Use when you want Impact AI to be the primary tracing backend

### Attach Mode

```python
init(mode="attach")
```

- Requires an existing TracerProvider
- Adds Impact AI as an additional exporter
- Raises `ValueError` if no TracerProvider exists

Use this when you have an existing OTel setup (Datadog, Honeycomb, etc.) and want to also send to Impact AI.

## BYO OpenTelemetry

Impact AI works alongside your existing observability stack:

```python
# Your existing OTel setup
from opentelemetry import trace
from opentelemetry.sdk.trace import TracerProvider
from opentelemetry.sdk.trace.export import BatchSpanProcessor
from opentelemetry.exporter.otlp.proto.grpc.trace_exporter import OTLPSpanExporter

# Set up your existing provider
provider = TracerProvider()
provider.add_span_processor(
    BatchSpanProcessor(OTLPSpanExporter(endpoint="your-otel-backend"))
)
trace.set_tracer_provider(provider)

# Now initialize Impact AI - it will attach to your existing provider
from impact.sdk import init
init(mode="auto")  # or mode="attach"
```

Traces will flow to both your existing backend and Impact AI.

## Batching Configuration

By default, the SDK batches spans for efficient export. Disable batching for debugging:

```python
init(disable_batch=True)
```

<Warning>
  Disabling batching increases network overhead. Only use for debugging.
</Warning>

## Log Capture

The SDK captures GenAI message content as logs by default. Disable if you don't want message content sent:

```python
init(disable_logs=True)
```

<Note>
  Disabling logs means you won't see input/output message content in traces, only metadata.
</Note>

## Shutdown

Gracefully shutdown the SDK to flush pending telemetry:

```python
from impact.sdk import shutdown

# At application exit
shutdown()
```

The shutdown function:
- Flushes all pending spans and logs
- Closes exporters cleanly
- Does NOT shut down user-provided TracerProviders (only Impact AI's components)

## Environment Variables

These environment variables affect SDK behavior:

| Variable | Description |
|----------|-------------|
| `IMPACT_API_KEY` | API key for authentication |
| `IMPACT_BASE_URL` | Impact AI endpoint URL |
| `OTEL_SEMCONV_STABILITY_OPT_IN` | Set automatically to `gen_ai_latest_experimental` |
| `OTEL_INSTRUMENTATION_GENAI_CAPTURE_MESSAGE_CONTENT` | Set automatically to `true` |

## Complete Example

```python
import os
import atexit
from impact.sdk import init, shutdown

# Initialize with explicit configuration
init(
    api_key=os.getenv("IMPACT_API_KEY"),
    api_endpoint=os.getenv("IMPACT_BASE_URL"),
    mode="auto",
    disable_batch=False,
    disable_logs=False,
)

# Register shutdown handler
atexit.register(shutdown)

# Your application code...
```

## ASGI Middleware

For web applications, wrap your ASGI app with OpenTelemetry middleware:

```python
from impact.sdk import init, instrument_asgi_app
from fastapi import FastAPI

init()

app = FastAPI()
app = instrument_asgi_app(app)
```

This creates root spans for HTTP requests, giving you end-to-end traces from HTTP request to LLM call.
