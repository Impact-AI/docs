Use `impact.init()` once at process startup.

## Signature

```python
impact.init(
    api_key: str | None = None,
    api_endpoint: str | None = None,
    mode: Literal["auto", "bootstrap", "attach"] = "auto",
    disable_batch: bool = False,
    disable_logs: bool = False,
)
```

## Parameters

| Parameter | Description |
|---|---|
| `api_key` | API key override (fallback: `IMPACT_API_KEY`) |
| `api_endpoint` | Endpoint override (typically only needed for self-hosted/custom endpoint routing) |
| `mode` | OpenTelemetry integration mode |
| `disable_batch` | Use simple processors instead of batch processors |
| `disable_logs` | Disable OTLP log pipeline |

## Mode Behavior

### `mode="auto"` (default)

- If a real tracer provider exists: attach Impact span processor/exporter
- Otherwise: bootstrap a provider

Best default for most apps.

### `mode="attach"`

- Requires an existing provider
- Does not replace the existing provider
- Raises `ValueError` if no provider exists

Use when you already own OTel setup and only want to add Impact export.

### `mode="bootstrap"`

- Always creates and sets a provider
- Impact becomes the primary provider for the process

## BYO OpenTelemetry Example

```python
from opentelemetry import trace
from opentelemetry.sdk.trace import TracerProvider

import impact

trace.set_tracer_provider(TracerProvider())

impact.init(mode="attach")
```

## ASGI Middleware

```python
from fastapi import FastAPI
import impact

impact.init()

app = FastAPI()
app = impact.instrument_asgi_app(app)
```

## Shutdown

```python
import atexit
import impact

impact.init()
atexit.register(impact.shutdown)
```

`shutdown()` flushes pending telemetry and shuts down Impact-owned processors/exporters.

## Notes

- SDK sets default GenAI semantic-convention env values if not already set.
- Calling `init()` multiple times is idempotent after the first successful init.
- For self-hosted deployments, you can set endpoint via `api_endpoint` or `IMPACT_BASE_URL`.
