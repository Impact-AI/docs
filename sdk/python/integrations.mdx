## Vector Databases

No vector database instrumentations are bundled with the Python SDK at this time. Vector store calls made through an instrumented framework (e.g. LangChain, LlamaIndex) are captured as part of that framework's spans.

## Disabling Specific Integrations

```bash
export OTEL_PYTHON_DISABLED_INSTRUMENTATIONS=openai,anthropic
```

<AccordionGroup>
  <Accordion title="Checking Instrumentation Status">
    ```bash
    export IMPACT_EXPORTER_DEBUG=1
    ```
  </Accordion>

  <Accordion title="Caveats">
    - `disable_logs=True` may reduce captured content — some instrumentors emit data via logs/events
    - Call `impact.init()` before importing `haystack` (detects tracing at import time)
    - If another library owns the `TracerProvider`, use `mode="auto"` or `mode="attach"`
    - Vertex AI instrumentation is included but deprecating
  </Accordion>

  <Accordion title="How Auto-Instrumentation Works">
    - `impact.init()` patches installed SDKs automatically
    - Frameworks emitting native OTel spans are exported as-is
    - No framework instrumentor? Model calls are still traced via the provider SDK
    - Add missing visibility with `@impact.trace`
  </Accordion>
</AccordionGroup>
