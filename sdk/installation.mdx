## Requirements

- Python 3.10+
- Network access from your runtime to your Impact OTLP endpoint

## Install

<CodeGroup>

```bash pip
pip install impact
```

```bash uv
uv add impact
```

```bash poetry
poetry add impact
```

</CodeGroup>

## Required Environment Variables

```bash
export IMPACT_API_KEY="impact_<region>_<token>"
```

<Note>
  For Impact Cloud, endpoint resolution is automatic from `IMPACT_API_KEY`.
  Use an explicit endpoint override only for self-hosted/custom deployments.
</Note>

## Optional: Self-Hosted Endpoint Override

```bash
export IMPACT_BASE_URL="https://api.<your-domain>"
```

## First Initialization

```python
import impact

impact.init()
```

Or explicit values (for self-hosted/custom endpoint):

```python
import impact

impact.init(
    api_key="impact_<region>_<token>",
    api_endpoint="https://api.<your-domain>",
)
```

## Verify It Works

```python
import impact
from openai import OpenAI

impact.init()

client = OpenAI()
client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "health check"}],
)
```

You should see traces in Analyse within seconds.

## Troubleshooting

<AccordionGroup>
  <Accordion title="Missing api_key error">
    Set `IMPACT_API_KEY` or pass `api_key=` explicitly to `impact.init()`.
  </Accordion>
  <Accordion title="Could not resolve endpoint from API key">
    Use an `impact_<region>_*` key format, or set `IMPACT_BASE_URL` for self-hosted/custom routing.
  </Accordion>
  <Accordion title="No traces arriving">
    Verify outbound access to `<endpoint>/v1/traces` and check auth key scope/version.
  </Accordion>
</AccordionGroup>
