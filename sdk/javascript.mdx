Use the official JS/TS SDK package: `impact-ai`.

## Install

```bash
npm i impact-ai
```

## Quickstart

```typescript
import impact from "impact-ai";
import OpenAI from "openai";

impact.init({
  apiKey: process.env.IMPACT_API_KEY,
  endpoint: process.env.IMPACT_BASE_URL, // optional for impact_<region>_* keys
});

impact.context({
  userId: "user_123",
  interactionId: "interaction_456",
  versionId: "v1.0.0",
  attributes: { team: "growth" },
});

const client = new OpenAI({
  apiKey: process.env.OPENAI_API_KEY,
});

const response = await client.chat.completions.create({
  model: "gpt-4o-mini",
  messages: [{ role: "user", content: "Say hello in one sentence." }],
});

console.log(response.choices[0]?.message?.content);
await impact.shutdown();
```

## Startup Model

1. Import `impact-ai` and call `impact.init(...)` first.
2. Import provider SDKs directly (`openai`, `@openai/agents`, `@google/genai`, etc.).
3. Keep `impact.init(...)` at startup so instrumentation attaches before first provider call.

## Public API

- `init(options?)`
- `context(ctx)`
- `trace(...)`
- `flush()`
- `shutdown()`
- `instrumentationResults`
- `vercelAITelemetry(options?)`

## Runtime Modes

| Mode | Behavior |
|---|---|
| `auto` | Attach when possible, otherwise bootstrap |
| `bootstrap` | Always create/register Impact providers |
| `attach` | Require existing provider, never replace it |

## Endpoint Resolution

1. `init({ endpoint })`
2. `IMPACT_BASE_URL`
3. Derived from `impact_<region>_*` API key

## Next.js

```typescript
// instrumentation.ts
import impact from "impact-ai";

export async function register() {
  impact.init({
    apiKey: process.env.IMPACT_API_KEY,
    endpoint: process.env.IMPACT_BASE_URL,
  });
}
```

## Vercel AI SDK

```typescript
import { generateText } from "ai";
import { openai } from "@ai-sdk/openai";
import impact from "impact-ai";

await generateText({
  model: openai("gpt-4.1-mini"),
  prompt: "hello",
  experimental_telemetry: impact.vercelAITelemetry(),
});
```

<Note>
  `impact.init(...)` must run before first AI SDK usage.
  Add `experimental_telemetry: impact.vercelAITelemetry()` to each traced AI SDK call.
</Note>

## Diagnostics

```typescript
impact.init({ diagLogLevel: "warn" });
```

```bash
export IMPACT_DIAG_LOG_LEVEL=warn
```

## Validate Instrumentation

```typescript
impact.init();

```
