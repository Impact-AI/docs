Simulations are automated test suites that systematically evaluate your AI product by running predefined scenarios and measuring performance.

## What Are Simulations?

Simulations combine:
- **Synthetic users** — AI-generated test users
- **Test scenarios** — Predefined interaction patterns
- **Evaluation** — Automated quality assessment
- **Analysis** — Results aggregation and insights

## Creating Simulations

### Via Iris Agent

```
You: Create a simulation suite for our support bot

Iris: I'll create a comprehensive simulation suite.

**Simulation: Support Bot Quality**

**Components:**
1. **Happy Path Tests** (100 scenarios)
   - Common questions answered correctly
   - Standard workflows completed

2. **Edge Case Tests** (50 scenarios)
   - Unusual inputs
   - Complex multi-turn conversations
   - Off-topic handling

3. **Persona Tests** (5 personas × 20 scenarios)
   - New users
   - Frustrated customers
   - Technical users
   - Casual browsers
   - VIP customers

4. **Adversarial Tests** (30 scenarios)
   - Prompt injection attempts
   - Inappropriate requests
   - Out-of-scope questions

**Total: 280 scenarios**

Ready to run?
```

### Configuration Options

| Option | Description | Default |
|--------|-------------|---------|
| Scenarios | Number of test scenarios | 100 |
| Personas | User personas to include | All |
| Topics | Topics to cover | All |
| Turns | Conversation length range | 1-5 |
| Parallel | Concurrent executions | 10 |

## Simulation Types

### Regression Simulation

Tests that same-version quality is maintained:

```
You: Run regression tests

Iris: Running regression simulation...

**Test Suite:** Standard Regression (v1.5.0)
**Baseline:** Previous run (Jan 25, 2025)

**Progress:** ████████████████████ 100%

**Results:**

| Metric | Baseline | Current | Δ |
|--------|----------|---------|---|
| Avg Score | 4.2 | 4.1 | -2.4% |
| Pass Rate | 92% | 90% | -2% |
| Edge Cases | 85% | 83% | -2% |

**Status:** ⚠ Minor Regression Detected

**Details:**
- 3 previously passing scenarios now failing
- All in "technical questions" category

Investigate failing scenarios?
```

### Stress Simulation

Test under high load:

```
You: Run stress test

Iris: Running stress simulation...

**Configuration:**
- Concurrent users: 100
- Duration: 10 minutes
- Ramp-up: 2 minutes

**Progress:** ████████████████████ 100%

**Results:**

| Metric | Baseline | Stress | Δ |
|--------|----------|--------|---|
| Latency (p50) | 1.2s | 1.8s | +50% |
| Latency (p99) | 2.1s | 5.2s | +148% |
| Error Rate | 0.1% | 2.3% | ⚠ |
| Quality Score | 4.2 | 4.0 | -5% |

**Observations:**
- Quality degrades under load
- Error rate spikes at >80 concurrent users
- p99 latency exceeds SLA at peak

**Recommendation:**
Consider scaling infrastructure before 
high-traffic periods.
```

## Best Practices

<AccordionGroup>
  <Accordion title="Run before deployments">
    Always run simulations before:
    - Production deployments
    - Major prompt changes
    - Model updates
    - Configuration changes
  </Accordion>
  
  <Accordion title="Maintain scenario diversity">
    Include scenarios for:
    - All major use cases
    - All user personas
    - Edge cases and errors
    - Adversarial inputs
  </Accordion>
  
  <Accordion title="Track trends">
    Monitor simulation results over time:
    - Establish baselines
    - Detect regressions
    - Measure improvements
  </Accordion>
  
  <Accordion title="Act on failures">
    When simulations fail:
    - Investigate root cause
    - Fix identified issues
    - Add to regression suite
    - Re-run to verify
  </Accordion>
</AccordionGroup>
