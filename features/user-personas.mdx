User personas are detailed profiles representing specific user types. They enable targeted testing of how your AI handles different user segments.

## What Are User Personas?

Personas are archetypal user profiles that capture:
- Demographics and background
- Communication patterns
- Needs and expectations
- Behavioral characteristics

## Creating Personas

### Via Iris Agent

```
You: Create a persona for enterprise customers

Iris: Creating "Enterprise Customer" persona:

**Profile:**
- Role: IT Manager or Technical Lead
- Company: 500+ employees
- Technical level: High
- Decision maker: Yes

**Communication Style:**
- Professional and formal
- Expects detailed technical answers
- Values precision over brevity
- May involve procurement questions

**Expectations:**
- SLA and uptime information
- Security and compliance details
- Integration capabilities
- Scalability information

**Example Questions:**
1. "What's your uptime SLA for enterprise tier?"
2. "Do you have SOC 2 certification?"
3. "How does your API handle rate limiting?"

Create test scenarios with this persona?
```

### Manual Configuration

Create personas with specific attributes:

```json
{
  "name": "Enterprise Customer",
  "description": "IT decision maker at large company",
  "attributes": {
    "role": "IT Manager",
    "company_size": "500+",
    "technical_level": "high",
    "industry": "financial_services"
  },
  "communication": {
    "style": "formal",
    "detail_level": "high",
    "patience": "moderate"
  },
  "topics": ["security", "compliance", "integration", "pricing"],
  "expectations": ["accurate", "detailed", "professional"]
}
```

## Persona Library

### Built-in Personas

Impact AI includes common personas:

| Persona | Description | Key Traits |
|---------|-------------|------------|
| **New User** | First-time visitor | Needs guidance, basic questions |
| **Power User** | Experienced, frequent | Advanced features, shortcuts |
| **Frustrated User** | Has unresolved issue | Impatient, needs resolution |
| **Technical User** | Developer or IT | Detailed technical questions |
| **Executive** | Decision maker | ROI, high-level overview |
| **Casual Browser** | Just exploring | General questions, low commitment |

### Custom Personas

Create personas specific to your product:

```
You: Create a persona for healthcare professionals

Iris: Creating "Healthcare Professional" persona:

**Profile:**
- Role: Doctor, Nurse, or Medical Admin
- Setting: Hospital, clinic, private practice
- Time pressure: High
- Compliance awareness: Critical

**Communication:**
- Direct and efficient
- Medical terminology expected
- Privacy-conscious
- Needs actionable information

**Key Concerns:**
- HIPAA compliance
- Patient safety
- Time efficiency
- Accuracy critical

**Test Scenarios:**
1. Quick lookup during patient consultation
2. Compliance-related questions
3. Emergency information needs
4. Administrative queries
```

## Using Personas for Testing

### Targeted Tests

Test specific personas:

```
You: Run tests with the "Frustrated User" persona

Iris: Running frustrated user tests...

**Scenario Types:**
- Repeat complaint
- Long wait frustration
- Escalation request
- Competitor comparison

**Results:**

| Scenario | Score | Notes |
|----------|-------|-------|
| Repeat complaint | 3.8/5 | Good acknowledgment |
| Long wait | 4.2/5 | Appropriate empathy |
| Escalation | 3.2/5 | ⚠ Slow to offer human |
| Competitor | 4.0/5 | Professional response |

**Recommendation:**
Improve escalation triggers for frustrated users.
They should be offered human agent sooner.
```

### Cross-Persona Comparison

Compare performance across personas:

```
Iris: Cross-Persona Analysis

| Persona | Avg Score | Weak Area |
|---------|-----------|-----------|
| New User | 4.3 | None |
| Power User | 4.1 | Feature depth |
| Frustrated | 3.6 | Empathy timing |
| Technical | 3.9 | Complex queries |
| Executive | 4.0 | Brevity |

**Insight:**
Frustrated users score lowest. Focus on:
1. Earlier empathy acknowledgment
2. Faster escalation paths
3. Proactive resolution offers
```

## Persona-Based Evaluation

### Module Configuration

Configure modules to evaluate persona-specific criteria:

```json
{
  "module": "Empathy Appropriateness",
  "persona_weights": {
    "frustrated_user": 2.0,
    "new_user": 1.5,
    "power_user": 0.8,
    "technical_user": 0.5
  }
}
```
