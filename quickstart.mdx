This guide gets you from a new account to live traces in a few minutes.

## Prerequisites

- Python 3.10+
- An app that calls at least one LLM provider (for example `openai`, `anthropic`, `google-genai`, or Bedrock)
- An Impact AI account at [beta.impact.ai](https://beta.impact.ai)

## Step 1: Sign Up and Create a Product Version

1. Open [beta.impact.ai](https://beta.impact.ai)
2. Complete onboarding and create your workspace
3. Create your product
4. Create a version for that product

Each API key is scoped to a product version.

## Step 2: Create an API Key

1. Go to **Settings -> API Keys**
2. Create a new key for your version
3. Copy the key (you will not be able to recover it later)

Set the key in your shell:

```bash
export IMPACT_API_KEY="impact_<region>_<token>"
```

<Note>
  For Impact Cloud, `IMPACT_API_KEY` is usually enough and endpoint routing is automatic from the key format.
  For self-hosted deployments, set `IMPACT_BASE_URL` (or pass `api_endpoint`) to your own ingest endpoint.
</Note>

## Step 3: Install the Python SDK

<CodeGroup>

```bash pip
pip install impact
```

```bash uv
uv add impact
```

```bash poetry
poetry add impact
```

</CodeGroup>

## Step 4: Initialize SDK at Startup

```python
import impact
from openai import OpenAI

impact.init()

client = OpenAI()

response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "Say hello in one line."}],
)

print(response.choices[0].message.content)
```

## Step 5: Add Request Context

```python
import impact

impact.context(
    user_id="user_123",
    interaction_id="conv_456",
    customer_tier="enterprise",
)
```

All spans created after this call in the current execution context carry `impact.context.*` attributes.

## Step 6: Add Manual Trace Spans (Optional)

```python
from impact import trace

@trace(type="workflow", name="chat_handler")
def handle_chat(message: str) -> str:
    return message.upper()
```

## Step 7: Verify in Impact AI

1. Open your product in the dashboard
2. Go to **Analyse -> Datasets** or **Analyse -> Monitor**
3. Confirm traces are arriving

## What Is Captured Automatically

- Model request and response payload metadata
- Tokens, latency, and timing
- Errors and exception details
- Tool and framework spans where instrumentation is available
