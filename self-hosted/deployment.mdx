This guide follows the current `infrastructure` repo workflow.

## Phase 1: One-Time Core Setup

Run once per organization to create shared resources.

### 1) Load Core Environment Variables

```bash
set -a; source core/.env.core; set +a
```

### 2) Authenticate

```bash
az login --tenant "$IMPACT_TENANT_ID"
az account set --subscription "$IMPACT_SUBSCRIPTION_ID"
gh auth login
gh auth refresh -h github.com -s admin:org
```

### 3) Deploy Core Resource Group + Bicep

```bash
az group create --name "$CORE_RG" --location "$CORE_LOCATION"

az deployment group create \
  --resource-group "$CORE_RG" \
  --name "main-$(date +%Y%m%d-%H%M%S)" \
  --template-file core/core.bicep \
  --parameters core="$CORE_NAME" domain="$CORE_DOMAIN" entraTenant="$ENTRA_TENANT"
```

### 4) Deploy Entra Tenant Artifacts

```bash
az login --tenant "$ENTRA_TENANT_DOMAIN" --allow-no-subscriptions
az rest -m POST -u "https://management.azure.com/providers/Microsoft.Authorization/elevateAccess?api-version=2016-07-01"
az role assignment create \
  --assignee-object-id "$(az ad signed-in-user show -o tsv --query id)" \
  --assignee-principal-type User \
  --role Owner \
  --scope /

az deployment tenant create \
  --name "entra-$(date +%Y%m%d-%H%M%S)" \
  --location "$CORE_LOCATION" \
  --template-file core/entra.bicep
```

### 5) Run Bootstrap Finalization

```bash
core/bootstrap.sh
```

## Phase 2: Post-Core Actions

After bootstrap:

1. Delegate DNS NS records to Azure DNS for your domain.
2. Publish ACS verification records for `notify.<domain>`.
3. Confirm ACS verification status is `Verified` before invite emails.
4. Store required integration secrets in Key Vault.
5. Deploy Marketplace webhook if used (`core/webhook/deploy.sh`).

## Phase 3: Provision an Environment

Create per-environment infrastructure (dev, prod, customer).

### GitHub Workflow (Recommended)

Use **Provision Environment** workflow in the `infrastructure` repo.

Inputs typically include:

- `environment` (for example `dev`, `acmecorp`)
- `subscription_id`
- `resource_group`
- `location`

### Manual CLI Path

```bash
export ENVIRONMENT=dev
export SUBSCRIPTION_ID=<subscription-id>
export RESOURCE_GROUP=rg-impact-$ENVIRONMENT
export LOCATION=swedencentral

az login
az account set --subscription "$SUBSCRIPTION_ID"
az group create --name "$RESOURCE_GROUP" --location "$LOCATION"

az deployment group create \
  --resource-group "$RESOURCE_GROUP" \
  --template-file platform/main.bicep \
  --parameters environment="$ENVIRONMENT" location="$LOCATION"

./platform/finalize.sh
```

## Phase 4: Deploy Applications

Run **Deploy Applications** workflow for the target environment after provisioning.

Expected endpoints:

- App URL: `https://<environment>.impact.ai`
- Telemetry ingest base: `https://api.<environment>.impact.ai`

## Operational Notes

- Core scripts are idempotent and safe to rerun.
- Environment reprovisioning may require deactivating existing Container App revisions first.
- Keep Core and environment configs in App Configuration/Key Vault synchronized.
