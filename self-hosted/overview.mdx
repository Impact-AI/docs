Impact AI self-hosting is Azure-first and follows a two-layer model:

1. **Core (shared, one-time)**
2. **Environment (repeatable per deployment: dev/prod/customer)**

## Deployment Layers

```text
Core (once)
  - Front Door, ACR, DNS, App Config, Key Vault, Entra External ID, shared comms

Environment (per instance)
  - VNet/private endpoints, Container Apps, Postgres/Redis/Storage, AI services
```

## Source of Truth Repos

- `infrastructure` repo: provisioning code (`core/`, `platform/`, `channels/`)
- `evaluation` repo: backend services
- `impact` repo: frontend applications

## What Core Provisioning Creates

From the current `core` templates/scripts, core setup provisions shared resources such as:

- Azure Front Door Premium + WAF policy
- Azure Container Registry
- Azure DNS zone
- App Configuration + Key Vault
- Azure Communication Services (email domain + sender)
- Entra External ID tenant and app registrations
- Core service principal for GitHub OIDC workflows

## Why This Model

- Avoids recreating global/shared infra per environment
- Keeps environment deployment repeatable and safer
- Supports SaaS, Lighthouse, and Marketplace channels on one codebase

## Requirements

- Azure subscription access (Owner-level for initial core operations)
- Azure CLI authenticated to the right tenant/subscription
- `gh` CLI for workflow/variable operations
- `jq`
